# HomeScreen 생리 상태 카드 UX 재설계

> 작성일: 2026-05-13 | 상태: 초안 v1.0

---

## 1. 배경 및 문제 정의

### 현재 구현 (`생리가 시작됐나요?` 카드)

**비생리 중 상태**
- 제목: "생리가 시작됐나요?" (질문형)
- 시작일 날짜 칩 가로 스크롤 (오늘 ~ 6일 전)
- 출혈량 선택 칩 (가벼움 / 보통 / 많음)
- "생리 시작" CTA 버튼

**생리 중 상태**
- 제목: "생리 중"
- 시작일 · 예상 종료일 텍스트
- 종료일 날짜 칩 가로 스크롤
- "생리 종료" CTA 버튼

### 문제점

| # | 문제 | 영향 |
|---|------|------|
| P1 | 질문형 제목이 데이터 컨텍스트 없이 등장 | 앱이 차갑고 폼(form) 느낌 |
| P2 | 날짜 칩 가로 스크롤이 카드 공간의 대부분 차지 | 시각적 노이즈, 핵심 정보 희석 |
| P3 | 생리 중 "몇 일째"인지 즉각적으로 파악 불가 | 사용자가 직접 계산해야 함 |
| P4 | 비생리 중일 때 예측 정보(예: D-5) 미노출 | 컨텍스트 없는 단순 입력 폼처럼 보임 |
| P5 | 생리 시작/종료 날짜 수정이 카드 내부에서만 가능 | 소급 수정 UX가 불편함 |

---

## 2. 목표

- **상태 중심 (Status-first)**: 사용자가 앱을 열었을 때 "지금 나는 어떤 단계인가"를 0.5초 안에 파악
- **액션은 보조**: 시작/종료 기록은 주인공이 아닌 보조 동작
- **공감형 UX**: 건강 앱답게 차갑지 않고, Luna가 맥락을 이해하는 느낌

## 3. 재설계 스펙

### 3-1. 비생리 중 상태 (예측 기반)

```
┌────────────────────────────────────────┐
│  🌙  생리 예정일까지  D-5              │
│      5월 18일 (월) 예정               │
│                                        │
│  ┌──────────────────────────────────┐  │
│  │      오늘 생리가 시작됐어요       │  │
│  └──────────────────────────────────┘  │
└────────────────────────────────────────┘
```

**규칙**
- 예측 데이터 있음: `생리 예정일까지 D-N` + 날짜 표시
- 예측 데이터 없음(첫 사용자): `생리 주기를 기록해보세요` + 서브텍스트 `시작일을 기록하면 예측이 시작돼요`
- CTA 문구: "오늘 생리가 시작됐어요" (능동형, 날짜 선택 없이 바로 오늘로 기록)
- 날짜 수정이 필요한 경우: CTA 아래 `다른 날짜로 기록 →` 텍스트 링크 → 바텀시트

### 3-2. 생리 중 상태

```
┌────────────────────────────────────────┐
│  🔴  생리 중  ·  3일째                 │
│      5/10 시작  ·  예상 종료 5/15     │
│                                        │
│  ████████████░░░░░░  3 / 5일          │
│                                        │
│  ┌──────────────────────────────────┐  │
│  │           생리 종료 기록          │  │
│  └──────────────────────────────────┘  │
└────────────────────────────────────────┘
```

**규칙**
- 타이틀: `생리 중 · N일째` (시작일로부터 경과일 실시간 계산)
- 서브텍스트: `시작일 · 예상 종료일` (예상 종료 = started_on + avg_period_length - 1)
- 프로그레스 바: `경과일 / 예상 기간` 비율, 넘으면 100% 고정
- CTA: "생리 종료 기록" → 탭 시 종료일 선택 바텀시트 열림
- 날짜 수정: CTA 아래 `시작일 수정 →` 텍스트 링크

### 3-3. 공통 바텀시트 (날짜 선택)

- 시작일 기록 / 시작일 수정 / 종료일 기록에 공통으로 사용
- 날짜 칩 가로 스크롤 (오늘 포함 7일) + 출혈량 선택 (시작 시에만)
- 확인 버튼으로 저장

## 4. 상태별 분기 로직

```
latestCycle 없음 OR (latestCycle 있고 ended_on != null)
  → 비생리 중 카드
    └─ prediction 있음  → D-N 표시
    └─ prediction 없음  → 안내 문구

latestCycle 있고 ended_on == null
  → 생리 중 카드
    └─ dayCount = today - started_on + 1
    └─ estimatedEnd = started_on + user.period_length_default - 1
    └─ progress = dayCount / user.period_length_default (max 1.0)
```

## 5. API 변경 사항

### 변경 없음 (기존 API 재사용)
- `POST /api/v1/cycles` — 생리 시작 (started_on, flow_level)
- `PATCH /api/v1/cycles/:id` — 생리 종료 (ended_on)
- `GET /api/v1/cycles?page=1&per=1` — 최신 주기 조회

### 필요 시 추가 고려
- `PATCH /api/v1/cycles/:id` 에 `started_on` 수정 허용 여부 검토
  - 현재 `update_params`는 `ended_on, flow_level` 만 허용
  - 시작일 수정 기능 구현 시 `started_on` 추가 필요

## 6. 컴포넌트 구조

```
HomeScreen
└── PeriodStatusCard (현재 periodCard)
    ├── InactivePeriodCard  (비생리 중)
    │   ├── PeriodCountdownLabel  (D-N 또는 안내 문구)
    │   └── StartPeriodButton
    └── ActivePeriodCard  (생리 중)
        ├── PeriodDayCounter  (N일째)
        ├── PeriodProgressBar
        └── EndPeriodButton
└── PeriodDateSheet (바텀시트 — 날짜 + 출혈량 선택)
```
