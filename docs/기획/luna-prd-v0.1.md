# Luna 서비스 기획서 v0.1

> 작성일: 2026-05-12 | 상태: 초안

---

## 1. 서비스 개요

**Luna**는 생리주기·배란일·증상을 추적하고, AI가 사용자의 데이터를 기반으로 건강 인사이트를 제공하는 iOS 앱입니다.

기존 주기 앱(Clue, Flo, Natural Cycles)과의 핵심 차별점은 **내 데이터를 이해하는 AI 어시스턴트**입니다. 단순 통계 수치 대신, 현재 주기 컨텍스트를 아는 AI와 대화하며 몸의 패턴을 이해할 수 있습니다.

---

## 2. 타겟 사용자

- **주 타겟**: 20-35세 여성, 스마트폰 활용에 익숙하고 건강 관리에 관심 있음
- **서브 타겟**: 임신을 계획하거나 피임 주기법에 관심 있는 여성
- **페르소나**:
  - 김지연(27세): 월경 불순으로 주기를 기록하고 싶지만 기존 앱의 복잡한 UI가 불편함
  - 이수아(31세): 임신을 준비 중. 배란일을 정확히 파악하고 싶고, 증상 변화의 의미를 이해하고 싶음

## 3. 핵심 기능 — Phase 1 MVP

### 3-1. 주기 기록
- 생리 시작일/종료일 기록
- 출혈량 (가벼움/보통/많음)
- 주기 자동 계산 및 이력 관리

### 3-2. 예측
- 다음 생리 예정일 (최근 6주기 가중평균)
- 배란 예정일 (다음 생리일 - 14일)
- 임신 가능 기간 (배란일 ±2일)
- 주기 데이터 3개 미만이면 기본값 28일 사용
- BBT 급등 / LH surge 기록 시 실측 배란일로 보정

### 3-3. 일일 증상 기록
- 증상 체크리스트: 경련/두통/피로/복부팽만/기분(5단계)/분비물 타입
- BBT(기초체온) 입력
- LH 테스트 결과 (양성/음성/미측정)
- **AI 자연어 입력**: "오늘 배가 아프고 기분이 별로야" → 자동 파싱

### 3-4. 캘린더 뷰
- 월간 달력, 주기 단계별 색상 코딩
  - 생리기: 테라코타
  - 가임기: 세이지 그린
  - 배란일: 앰버
  - 황체기: 기본
- 날짜 탭 → 해당 날 증상 요약

### 3-5. AI 어시스턴트 ⭐ 차별화
- **증상 분석 채팅**: 현재 주기 컨텍스트 기반 Q&A
  - "지금 두통이 왜 생기나요?" → 황체기 프로게스테론 영향 설명
  - "임신 가능성이 있나요?" → 주기 데이터 기반 안내
- **월간 자연어 리포트**: 매달 말 주기 패턴을 읽기 쉬운 문장으로 요약
- 모델: Claude Haiku 3.5 (Prompt Caching 적용)
- 개인정보 최소 전송: 날짜/증상 코드만 전송, 사용자 식별 정보 제외

### 3-6. 인사이트
- 평균 주기 길이 트렌드
- BBT 차트 (라인 그래프)
- 증상 빈도 히트맵
- AI 월간 요약 카드

## 4. 비기능 요구사항

### 보안 / 개인정보
- 생리 데이터는 민감 개인정보 — 서버 저장 시 암호화 필수
- JWT 인증, HTTPS 전용
- AI 전송 시 사용자 식별 정보(이름/이메일/ID) 미포함
- 앱 잠금 (생체인증 / PIN) 지원

### 성능
- 달력 로딩 < 300ms
- AI 응답 스트리밍 (첫 토큰 < 1.5초)

### 플랫폼
- iOS 16+ 우선 (Android Phase 2)
- Expo + React Native

## 6. API 설계 요약

| 엔드포인트 | 설명 |
|---|---|
| `POST /auth/signup` | 회원가입 |
| `POST /auth/login` | 로그인 → JWT 발급 |
| `GET/POST /cycles` | 주기 목록 조회/기록 |
| `PUT /cycles/:id` | 주기 종료일·출혈량 수정 |
| `GET/POST /daily_logs` | 일일 증상 조회/기록 |
| `GET /predictions/current` | 현재 주기 예측 데이터 |
| `POST /ai/chat` | AI 증상 분석 (스트리밍) |
| `GET /ai/monthly_report` | 월간 자연어 리포트 |

## 8. 알림 전략

### 알림 종류 및 타이밍

| 알림 | 트리거 | 기본값 |
|---|---|---|
| 생리 예정 알림 | 예정일 D-3, D-1 | 오전 9시 |
| 배란 예정 알림 | 예정 배란일 D-2, D-0 | 오전 9시 |
| 가임기 시작 알림 | 가임기 첫날 | 오전 9시 |
| 생리 시작 기록 독려 | 예정일 당일 미기록 시 | 오후 8시 |
| 일일 로그 리마인더 | 매일 (선택적, 기본 off) | 오후 10시 |
| 월간 리포트 준비됨 | 주기 종료 후 | 오전 10시 |

### 구현 방식
- iOS: `UNUserNotificationCenter` (Local Notification 우선, Push는 Phase 2)
- 알림 권한 요청: 온보딩 완료 직후 (이유 설명 후 요청)
- 모든 알림 시간·종류는 설정에서 개별 on/off 가능
- 예측이 없는 경우 (데이터 부족) 알림 발송 안 함
